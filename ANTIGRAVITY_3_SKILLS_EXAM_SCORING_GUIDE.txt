1.1 Current exam generation
---------------------------
Current route:
src/routes/exam.js

Current flow:
1. POST /api/exam/start validates name/email/position/consent.
2. positionInfo(position) maps to BANK_STAFF or BANK_OFFICE_MGR.
3. getBank(posInfo.bank) loads bank.
4. If exam_configs has fixed config, selected audio/passage/writing are used.
5. Otherwise sampleQuestions(bank, posInfo.management) is used.

     writing: [id...]
   }
7. shieldForClient(questionSet) strips correct answers before returning to frontend.

Current sampleQuestions behavior:
- Listening: picks 2 audio groups.
- Reading: picks 1 passage group.
- Writing: picks 5 random/deduplicated writing items.

Current weakness:
- No formal exam blueprint.
- No versioned exam plan.
- No target score weights per skill except implicit /10 each.
- No item quality metadata.
- No reliable skill/subskill coverage.
- No clear mapping from selected items to CEFR evidence.

-------------------
Current scorer:
src/lib/scoring.js

Current official score:
- Listening: /10
- Reading: /10
- Writing: /10
- Total: /30

Current listening/reading scoring:
- MCQ only.
- correctCount / questionCount * 10, rounded.

Current writing scoring:
- deterministic types:
  - fill_blank
  - error_correction
  - sentence_order
  - sentence_transform
- short_answer uses AI grader if available.
- if AI fails, short_answer falls back to word count:
  - >= minWords => 0.8 item score
  - > 0 words  => 0.4 item score

Critical weakness:
The length-based writing fallback must not be used for official scoring. Word count alone is not valid evidence of English proficiency.

4. Feature Flags
================

Add these env variables:

EXAM_ENGINE_VERSION=v1
SCORING_VERSION=v1
SCORING_SHADOW_MODE=false
WRITING_OFFICIAL_MODE=deterministic
ALLOW_AI_OFFICIAL_SCORING=false
ALLOW_LENGTH_BASED_WRITING_FALLBACK=false
OFFLINE_SCORING_MIN_CONFIDENCE=0.75
OFFLINE_SCORING_REVIEW_MARGIN=1.5
EXAM_BLUEPRINT_VERSION=vt_3skills_v1

Meanings:

EXAM_ENGINE_VERSION:
- v1: existing sampleQuestions() behavior.
- v2: new blueprint-based generator.

SCORING_VERSION:
- v1: existing scoreAnswers() behavior, but length fallback must be disabled by default.
- v2: new score engine.

SCORING_SHADOW_MODE:
- true: run v2 scoring and store details but keep v1 official response.
- false: only run selected scoring version.

WRITING_OFFICIAL_MODE:
- deterministic: only deterministic/rule-based writing contributes official score.
- review_required: writing free response always becomes pending_review.

ALLOW_AI_OFFICIAL_SCORING:
- default false.
- If true in a future experiment, AI score can be shown as advisory only unless explicitly approved.

ALLOW_LENGTH_BASED_WRITING_FALLBACK:
- default false.
- Never enable in production unless business explicitly accepts the risk.

-------------------
Target internal level:
- A2-B1 evidence.
- Staff pass target: B1 internal band.

Reading /10:

- 8-10 questions total.
- Mix:
  - main idea
  - detail
  - vocabulary in context
  - true/false/not given if available

Writing /10:
- 5 objective/controlled tasks.
- No AI official scoring.
- Recommended mix:
  - 1 fill_blank
  - 1 error_correction
  - 1 sentence_order
  - 1 sentence_transform
  - 1 controlled_response OR additional deterministic item

If controlled_response is not ready:
- Use only deterministic writing item types.

7.2 selector.js
---------------
Selection rules:

General:
- only use items with status missing or "active"
- prefer target levels from blueprint
- avoid duplicate content fingerprints
- preserve complete listening audio groups
- preserve complete reading passage groups
- enforce max/min question counts

Listening:
- group by audioGroupId or audioFile.
- select N audio groups per blueprint.
- include all items in selected groups, then cap if needed.
- if group has too many items, select by subskill coverage first.

Reading:
- group by passageId.
- select N passage groups.
- include all items in selected passage group unless too many.
- prefer passage groups matching target levels and subskill coverage.

Writing:
- select by typeTargets.
- if controlled_response not available, use fallbackTypeTargets.
- never select short_answer for official v2 unless explicitly allowed and it is review-required.

Fallback:
- If v2 cannot satisfy blueprint, return a clear error in development.
- In production, fallback to v1 only if EXAM_ENGINE_FALLBACK_TO_V1=true.
- Log exact missing coverage.

8. Scoring V2 Design
====================

8.1 Official score model
------------------------
Keep compatibility:

scores = {
  listening: 0-10 integer or one decimal,
  reading: 0-10 integer or one decimal,
  writing: 0-10 integer or one decimal
}

total = listening + reading + writing
max = 30

Recommended precision:
- per-item raw score: 0-1
- skill score: one decimal internally, but response can be rounded if frontend expects integer.
- Preserve current integer output if required.

Recommended:
- Use one decimal in details.
- Store rounded skill score in DB as current INTEGER columns if schema remains unchanged.


8.2 Listening scoring
---------------------
Input:
- questionSet.listening
- answers.listening

Algorithm:
1. For each item:
   - compare Number(userAnswer) === Number(q.correct)
   - itemScore = 1 or 0
2. raw = sum(itemScore * points) / sum(points)
3. skillScore = roundToOneDecimal(raw * 10)

Details:

{
  "id": "...",
  "skill": "listening",
  "subskill": "detail",
  "level": "B1",
  "userAnswer": 2,
  "correct": 1,
  "isCorrect": false,
  "pointsEarned": 0,
  "pointsMax": 1
}

8.4 Writing scoring
-------------------
Writing v2 has two categories:

A. Objective deterministic writing:
- fill_blank
- error_correction

- sentence_transform

B. Controlled response:

Official v2 must not use AI.
Official v2 must not use length-only scoring.


8.4.1 fill_blank
----------------
Current issue:
Only exact lowercase string match is supported.

Improve:
- support accepted alternatives:
  blanks: { "1": ["reservation", "booking"] }
- normalize:
  - trim
  - lowercase
  - collapse whitespace
  - strip harmless punctuation

Score:
- correct / total


8.4.2 error_correction
----------------------
Keep current approach:
- selected index equals s.correct.

Score:
- correct / total


8.4.3 sentence_order
--------------------
Current approach:
- position match only.

Improve optionally:
- position score: 70%
- adjacency score: 30%

For low-risk first implementation:
- keep position score only.
- add details for future adjacency score.


8.4.4 sentence_transform
------------------------
Current issue:
given.includes(acc) or acc.includes(given) can create false positives.

New rule:
- exact normalized match OR token similarity >= 0.85 for answers with >= 5 tokens.
- for accepted answers shorter than 5 tokens, require exact normalized match.

Return:
- acceptedMatched
- similarity
- correct boolean


8.4.5 controlled_response
-------------------------
This is the main replacement for free AI writing.

Rubric total: 100, normalized to 0-1.

Criteria:
- Task Fulfilment: 30
- Language Accuracy: 25
- Vocabulary & Domain Use: 20
- Coherence & Organization: 15
- Fluency & Range: 10

Task Fulfilment:
- requiredElements coverage.
- prompt relevance.
- format completion if email/service note.

Language Accuracy:
Use deterministic penalties only:
- too few valid English-like tokens
- no complete sentence
- excessive punctuation/noise
- repeated sentence
- common pattern errors if detectable

Vocabulary:
- domainTermCoverage.
- formal phrase usage.
- avoid keyword stuffing.

Coherence:
- greeting/closing for email.
- connector count.
- paragraph/sentence organization.

Fluency & Range:
- sentence count.
- average sentence length.
- unique token ratio.
- not all sentences identical/simple.

Confidence:
- calculated separately.
- low confidence triggers pending_review.

Do not overclaim grammar accuracy. Rule-based scorer is transparent and conservative.


8.5 Pending review rules
------------------------
Set flags.writing_pending_review = true if:

- controlled_response confidence < OFFLINE_SCORING_MIN_CONFIDENCE
- writing score is near pass/fail threshold
- answer is blank or too short
- answer copies prompt heavily
- answer appears non-English
- answer is suspicious/repetitive
- item type is short_answer and no deterministic rubric is available

When pending review:
- score_writing can be null or provisional.
- score_total must be null to preserve current pending_review behavior.
- status = pending_review.

Current admin pending-review route already supports manual finalize.
Do not break it.

async function scoreAnswers(questionSet, answers, options = {}) {
  const version = process.env.SCORING_VERSION || 'v1';
  const shadow = String(process.env.SCORING_SHADOW_MODE).toLowerCase() === 'true';

  if (version === 'v2') {
    return require('./scoring-v2').scoreAnswersV2(questionSet, answers, options);
  }

  const v1 = await scoreAnswersV1(questionSet, answers, options);

  if (shadow) {
    try {
      v1.shadow = await require('./scoring-v2').scoreAnswersV2(questionSet, answers, options);
      v1.flags = v1.flags || {};
      v1.flags.shadow_scoring_v2 = true;
    } catch (err) {
      v1.flags = v1.flags || {};
      v1.flags.shadow_scoring_v2_error = err.message;
    }
  }

  return v1;
}

Also change v1 short_answer behavior:
- default must not length-score.
- if AI fails and ALLOW_LENGTH_BASED_WRITING_FALLBACK !== "true":
  flags.writing_pending_review = true
  details.writing.push({ id, type, pending: true, reason: "grader_unavailable" })
